# OKR数据监测看板

## 项目简介
这是一个用于监控OKR数据可视化的看板系统，支持多维度指标展示、数据管理和报告导出功能。

## 在线访问
[点击访问看板](https://your-username.github.io/okr-dashboard/)

## 团队协作流程

### 角色分工
- **数据管理员**：负责每月更新数据，导出报告
- **团队成员**：查看看板，接收报告

### 数据管理员的操作流程

#### 1. 每月数据更新
1. 访问看板页面，查看当前数据
2. 点击"管理数据"按钮进入管理页面
3. 选择需要更新的月份
4. 修改各项指标的实际值
5. 点击"保存数据"按钮（数据保存在浏览器本地）
6. 点击"导出报告"按钮生成月度报告

#### 2. 报告分享
1. 将导出的报告图片保存到本地
2. 通过邮件/钉钉/微信分享给团队成员
3. 报告格式：PNG图片，包含完整的月度数据

#### 3. 数据备份（可选）
如果需要长期保存数据：
- 在管理页面点击"导出"按钮导出JSON数据文件
- 将JSON文件保存到本地或共享文件夹
- 需要时可以重新导入数据

### 团队成员的使用方式

#### 查看看板
1. 访问在线看板地址
2. 查看各维度的整体进度
3. 查看详细的指标数据和趋势图
4. 查看汇总表格了解达成情况

#### 接收报告
1. 通过邮件/钉钉接收月度报告
2. 查看报告中的详细数据
3. 了解各指标的达成情况

## 数据更新说明

### 指标类型
- **正向指标**：数值越高越好（如：HC达成率、回款及时率）
- **反向指标**：数值越低越好（如：费用包使用率、骨干流失率）

### 数据更新时机
- 每月1-3号更新上个月的数据
- 确保所有指标数据准确无误
- 检查数据合理性后再保存

### 注意事项
- 数据保存在浏览器本地，清除缓存会丢失数据
- 建议定期导出JSON数据文件进行备份
- 导出的报告图片包含完整数据，可以长期保存

## 技术说明

### 技术栈
- 纯HTML/CSS/JavaScript
- Chart.js（图表库）
- html2canvas（截图导出）
- 无需后端服务器

### 部署方式
- GitHub Pages托管
- 自动部署，无需手动配置
- 支持HTTPS安全访问

### 数据存储
- 浏览器LocalStorage
- 支持数据导入导出
- 支持清除缓存重新加载

## 常见问题

### Q: 数据更新后其他人看不到怎么办？
A: 当前版本数据保存在浏览器本地，需要通过导出报告的方式分享。如需实时同步，需要部署后端服务器。

### Q: 如何恢复初始数据？
A: 点击看板页面右上角的"清除缓存"按钮，可以恢复到data.js中的初始数据。

### Q: 报告导出失败怎么办？
A: 检查浏览器是否允许下载文件，确保网络连接正常。如果仍然失败，尝试刷新页面后重新导出。

### Q: 如何修改指标配置？
A: 需要修改data.js文件，重新部署到GitHub Pages。建议由技术负责人进行修改。
